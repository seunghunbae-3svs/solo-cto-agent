# Review — Synthesis & Decision Framework

## Post-Lens Synthesis Template

After the three lenses (investor, user, competitor) run, synthesize into this structure:

```
OVERALL ASSESSMENT: [Score /10]

Strengths:
1. [Specific, evidence-backed statement]
2. [Specific, evidence-backed statement]

Critical gaps:
1. [What's missing or weak]
2. [What needs validation]

Contradictions:
1. [Where the three lenses disagree]
   Example: Investor thinks it's a $1B market, user feedback says niche.

Recommended changes:
1. [Actionable next step]
2. [Actionable next step]
3. [Actionable next step]
```

---

## Score Mapping (/10)

| Range | Meaning | Next step |
|---|---|---|
| 9-10 | Ship immediately. Pressure-testing passed; risks are known and manageable. | Build/deploy |
| 7-8 | Strong concept with fixable gaps. Iterate on 1–2 blocking assumptions. | Refine design or run pilot |
| 5-6 | Interesting but needs work. Multiple assumptions untested. | Validate via pilot/customer interviews |
| 3-4 | Fundamental flaw or unproven market. Pivoting or deep redesign needed. | Reconsider scope or market |
| 1-2 | Core assumption broken or cost/moat story collapses. | Likely not viable in current form |

---

## Common Contradiction Patterns

### Pattern 1: Investor loves it, User ignores it
**Signal**: High TAM and defensibility, but no strong unit-level appeal.
**Fix**: Either reframe value prop to match user pain, or find a smaller, more motivated user segment.

### Pattern 2: User wants it, Investor sees no moat
**Signal**: Real pain solved, but easily copyable.
**Fix**: Build defensibility (data, network, brand) while growing. Accept lower margins initially.

### Pattern 3: Competitor can neutralize it, but you move fast
**Signal**: Weak moat, but first-mover share capture.
**Fix**: Use early traction to build real moats (switching cost, data, community). Know your time window.

### Pattern 4: All three lenses positive, but numbers don't work
**Signal**: Great product-market fit, but margin is too thin or CAC is too high.
**Fix**: Fix unit economics before scaling. Raise prices, reduce COGS, or find higher-margin channels.

## When Synthesis Reveals "Don't Do This"

If synthesis lands at 1–2, communicate clearly:

```
OVERALL ASSESSMENT: 2/10

This is not a timing issue or a marketing problem.
The core assumption [X] is broken, and it makes the business unviable.

If you want to pursue [related idea], here's what would change the assessment:
1. ...
2. ...

Otherwise, this should be shelved or pivoted significantly.
```

Resist the urge to soften this. If it's broken, say so.

---

## Anti-Patterns in Synthesis

```
❌ Only listing strengths
   → Balance with critical gaps every time

❌ Being so polite that the real problem disappears
   → Name the problem directly

❌ Confusing possibility with evidence
   → Tag assumptions explicitly

❌ Ignoring business model risk
   → Always include unit economics or margin analysis

❌ Comparing only to weak competitors
   → Ask "What would [leader in space] do?"

❌ Treating first-mover status as a moat by default
   → Acknowledge it buys time, not defensibility

❌ Saying something is promising without saying what could break it
   → Always include "Would fail if..." clause
```
